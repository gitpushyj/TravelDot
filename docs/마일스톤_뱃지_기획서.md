# VisitGrid 마일스톤 & 뱃지 시스템 기획서

> 여행 기록을 정량화하고, 사용자에게 "내 위치"를 알려주는 게이미피케이션 레이어

---

## 1. 목적

현재 홈 화면의 "다음 마일스톤" 표시는 단순한 임계값 배열(`[5, 10, 15, 25, 50, 100, 150, 195]`)에 기반하고 있어, **사용자가 자기 수준을 객관적으로 가늠하기 어렵습니다.**

이 기획서는 다음 세 가지를 정의합니다.

1. **통계 기반 10단계 등급 시스템** — 전 세계 인구 분포의 분위(decile)를 기준으로 사용자의 여행 경험을 절대적/상대적으로 평가
2. **상위 1% / 0.1% 명예 등급** — 일반 분위로는 표현 불가능한 극단 여행자에 대한 별도 등급
3. **확장 가능한 뱃지 체계** — 국가 수 외에도 여행 일수, 지역(대륙/하위 권역) 정복 등 다축(multi-axis) 성취 시스템

## 2. 통계 조사 요약

### 2.1 글로벌 평생 방문 국가 수 분포 (해외 기준 원자료)

> 외부 통계는 일반적으로 **해외 방문(international visits)** 기준이므로, 본 기획에서 "본국 포함" 카운트로 변환하려면 **모든 값에 +1** 을 적용한다.

| 분위 | 누적 인구 | 해외 방문국가 | 본국 포함 환산 | 출처 / 근거 |
|---|---|---|---|---|
| 0개국 (해외 무경험) | ~18% | 0 | 1 (본국만) | Agoda/YouGov 2019 |
| 1~10 | ~86% (누적) | 1~10 | 2~11 | Agoda/YouGov 2019 |
| 11~30 | ~96% (누적) | 11~30 | 12~31 | 동일 |
| 31+ | 상위 ~4% | 31~ | 32~ | 영국 5%, 호주 4%, 미국 2% |
| **상위 1%** | — | ~50 | ~51 | NomadMania 일반 회원대 |
| **상위 0.1%** | — | ~100 | ~100 | TCC 가입선 (본국 포함 100개국) |
| 상위 0.000065% | — | 192~193 | **193** | NomadMania UN Master ≈ 521명 (2025-12) |

> **상단 컷오프(100, 150, 193)는 본국이 포함된 절대 기준점**이라 +1을 추가하지 않는다 — 본국은 항상 UN 193개국 안에 포함되어 있기 때문.

### 2.2 국가별 평균 방문 국가 수 (해외 기준)

- **영국인** 평균 ~12개국, **호주인** ~10개국 (가장 많은 편)
- **미국인** 평균 ~3개국, 약 25%는 해외 경험 전무
- **일본/중국** 응답자 과반(54~57%)이 1~5개국
- **글로벌 평균**: 약 5개국 (단, 중앙값은 2~3개국)

본국 포함 시 모든 평균에 +1 → 영국 ~13, 호주 ~11, 미국 ~4, 글로벌 평균 ~6.

### 2.3 참고 단체

| 단체 | 자격 | 회원 수 |
|---|---|---|
| **TCC** (Travelers' Century Club) | 100개국 이상 | ~1,400~1,500명 |
| **NomadMania UN Master** | 193개국 모두 | ~521명 (검증) |
| **MTP** (Most Traveled People) | 자체 1,300+ 지역 리스트 | — |

- [Agoda Well-Traveled Survey 2019 (YouGov)](https://www.agoda.com/press/well-traveled-survey-2019/)

## 3. 10단계 등급 설계

### 3.1 설계 원칙

1. **분위 기반(percentile-based)** — 각 등급은 글로벌 인구 분포의 약 10%씩을 점유하도록 설계 (Decile)
2. **라운드 넘버 우선** — 엄격한 분위(예: 4.7개국)보다 직관적 컷오프를 사용. 통계적 정확도와 UX 가독성의 트레이드오프
3. **하위 분위 압축, 상위 분위 확장** — 인구의 ~86%가 (해외 기준) 10개국 이하에 몰려 있으므로 하위 6단계는 좁은 간격으로, 상위 4단계는 넓은 간격으로 배치
4. **본국(home country) 포함** — 등급 산정의 기반이 되는 "방문 국가 수"는 **본국을 포함한 평생 방문 국가 전체**를 집계한다. 본국 또한 살아오며 발자취를 남긴 "평생의 여행지"이며, GPS 사진으로 검증 가능하다는 동일한 데이터 신뢰성 기준을 만족한다.

### 3.2 "방문 국가 수"의 정의

> **`visitedCountries = 사진의 GPS EXIF로 검증된 고유 국가 코드 집합` (본국 포함)**

- 본국 미설정 사용자에게도 동일하게 적용 (본국 설정과 무관하게 사진의 GPS 코드만이 카운트의 원천)
- 통계 출처(Pew, Agoda)는 보통 "해외 방문"을 묻기 때문에, 본 기획의 컷오프는 **외부 통계 + 1** 로 환산하여 정렬됨 (단, 100·150·193 등 절대 상한 기준은 +1 환산 대상 아님 — §2.1 단서 참조)
- 본국 카운트의 트리거는 **본국에서 찍은 GPS 사진 1장 이상** — 사진이 없으면 카운트되지 않음. 즉 본국이라도 "기록"이 있어야 인정 (앱의 핵심 가치인 "기록 기반 수집"과 정합)

### 3.3 등급표

| Tier | 분위(인구 누적) | 컷오프 | 명칭 (KR / EN) | 의미 |
|---|---|---|---|---|
| **T1** | ~18% (하위) | **1개국** | 정착민 / **Settler** | 본국에만 발자취. 여정의 출발선 |
| **T2** | ~30% | **2개국** | 첫걸음 / **First Step** | 인생 첫 해외 여행 |
| **T3** | ~45% | **3개국** | 산책자 / **Stroller** | 거리에 얽매이지 않고 발걸음을 넓혀가는 |
| **T4** | ~58% | **4개국** | 입문 여행자 / **Wayfarer** | 글로벌 중앙값 부근 |
| **T5** | ~70% | **6개국** | 여행자 / **Traveler** | 글로벌 평균 도달 (해외 5 + 본국) |
| **T6** | ~80% | **9개국** | 탐험가 / **Explorer** | 영미권 평균에 근접 |
| **T7** | ~88% | **14개국** | 베테랑 / **Veteran** | 영국인 평균 상회 |
| **T8** | ~94% | **22개국** | 노마드 / **Nomad** | 상위 6% — 본격 여행 마니아 |
| **T9** | ~98% | **36개국** | 글로브트로터 / **Globetrotter** | 상위 2% — 진지한 수집가 |
| **T10** | ~99% | **50개국** | 월드 컬렉터 / **World Collector** | 상위 1% 진입선 (UN의 1/4) |

> T1~T9 컷오프는 외부 통계의 "해외 방문" 분위에 +1 한 값이며(예: 글로벌 평균 5 → 6), T10 이상은 절대 기준점이라 +1 적용 대상이 아님.

### 3.4 명예 등급 (Prestige Tiers)

T10 위로는 일반 인구 분위로 구분이 무의미하므로 **상위 0.x% 단위의 명예 등급**으로 취급합니다.

| 등급 | 컷오프 | 명칭 | 비고 |
|---|---|---|---|
| **★ Master** | **75개국** | 마스터 / **Master** | 상위 ~0.5% 추정 |
| **★★ Grandmaster** | **100개국** | 그랜드마스터 / **Grandmaster** | 상위 ~0.1% — TCC 자격선 (본국 포함 100) |
| **★★★ Legend** | **150개국** | 레전드 / **Legend** | TCC Silver급 — 추정 0.001% |
| **★★★★ UN Master** | **193개국** | UN 마스터 / **UN Master** | 본국을 포함한 UN 전 회원국 정복. 검증 인원 ~521명 (2025-12) |

> **표시 정책**: 홈 화면에는 항상 "현재 등급(T1~T10)"과 "다음 등급까지 남은 국가 수"를 노출. 명예 등급 도달자에 한해 별도 강조 UI(금색 테두리, 별 아이콘 등) 적용.

### 3.5 등급 간 거리(체감 난이도) 검증

| 구간 | 추가 필요 국가 수 | 체감 난이도 |
|---|---|---|
| T1 → T2 | 1 | 매우 쉬움 (생애 첫 출국) |
| T2 → T3 | 1 | 쉬움 |
| T3 → T4 | 1 | 쉬움 |
| T4 → T5 | 2 | 보통 |
| T5 → T6 | 3 | 보통 |
| T6 → T7 | 5 | 보통 |
| T7 → T8 | 8 | 어려움 |
| T8 → T9 | 14 | 매우 어려움 |
| T9 → T10 | 14 | 매우 어려움 |
| T10 → Master | 25 | 극도로 어려움 |
| Master → Grandmaster | 25 | 극도로 어려움 |
| Grandmaster → Legend | 50 | 극도로 어려움 |
| Legend → UN Master | 43 | 평생의 도전 |

**의도**: 초반에는 등급이 빠르게 오르며 성취감을 주고(T1 → T5는 본국 + 5개국이면 도달), 상위로 갈수록 지수적으로 어려워지는 RPG식 곡선을 따릅니다.

> **본국 카운트 시작 조건**: T1(1개국)은 "본국에서 찍은 GPS 사진 1장 이상" 필요. 따라서 T1조차도 "기록 0인 신규 사용자"는 아직 도달하지 못한 상태(0개국)로 표시 — 첫 사진 등록 직후 T1 잠금 해제 연출이 가능.

## 4. 뱃지 시스템 설계

### 4.1 뱃지의 두 종류

뱃지는 **등급 뱃지(Tier Badge)** 와 **테마 뱃지(Theme Badge)** 두 축으로 분리합니다.

```
[등급 뱃지]                [테마 뱃지]
T1 → T10 + Prestige 4종    국가 수와 별개로
(총 14종)                   "어디를 / 얼마나 다녔는가"를 표현
```

### 4.2 등급 뱃지 (Tier Badge) — v1 범위

- 14종(T1~T10 + Master/Grandmaster/Legend/UN Master)
- 사용자는 **현재 한 개**의 등급 뱃지를 보유 (시간이 지나며 갱신)
- 과거에 도달한 적 있는 등급은 "회고 뱃지(Achievement)"로 따로 보존 — 한 번 얻으면 사라지지 않음

#### 시각 디자인 가이드

| 등급군 | 색상 톤 | 모양 |
|---|---|---|
| T1~T3 | 무채색/Soft Gray | 작은 원형 — "출발선" 인상 |
| T4~T6 | Earth tone (모래/흙/올리브) | 5각/6각형 |
| T7~T9 | Cool tone (바다/하늘) | 방패형 — "베테랑" 인상 |
| T10 | Warm gold | 별형 |
| Master | Gold + 별 1개 | |
| Grandmaster | Platinum + 별 2개 | |
| Legend | Diamond + 별 3개 | |
| UN Master | Iridescent + 지구본 | 최상위 — 별도 애니메이션 |

### 4.3 테마 뱃지 (Theme Badge) — v2 이후 확장

#### 4.3.1 여행 일수 뱃지 (Day-based, v2 구현 범위)

총 누적 여행 일수(`visit_days` 테이블의 모든 행 수)에 따른 6단계 뱃지.

| Day Tier | 컷오프 | 명칭 (KR / EN) | 의미 |
|---|---|---|---|
| D1 | **7일** | 일주의 여행자 / **Weekender** | 첫 일주일 누적 |
| D2 | **30일** | 한 달의 방랑 / **Monthly Wanderer** | 한 달치 발자취 |
| D3 | **100일** | 백일의 여정 / **Centenary Voyager** | 한 자리 → 세 자리 진입 |
| D4 | **365일** | 일년의 방랑자 / **Yearling Drifter** | 1년 누적 |
| D5 | **730일** | 두 해의 떠돌이 / **Biennial Drifter** | 2년 누적 |
| D6 | **1,000일** | 영원한 여행자 / **Eternal Traveler** | 평생 컬렉션 |

> 본국 사진은 자동 동기화에서 제외되어 `visit_days`에 거의 들어오지 않으므로(§5의 `removeAutoVisitsForCountry` 정책), 일수는 **사실상 해외 체류일** 로 동작한다. 본국에서 수동 추가한 날도 있다면 이는 사용자가 의도적으로 기록한 것이므로 합산해도 의미가 흐려지지 않는다.

|---|---|---|
| 동아시아 | 7 | 동아시아 마스터 |
| 동남아시아 | 11 | 동남아시아 마스터 |
| 남아시아 | 9 | 남아시아 마스터 |
| 중동 | 18 | 중동 마스터 |
| 서유럽 | 9 | 서유럽 마스터 |
| 북유럽 | 10 | 북유럽 마스터 |
| 동유럽 | 10 | 동유럽 마스터 |
| 남유럽 | 16 | 남유럽 마스터 |
| 북아메리카 | 23 | 북미 마스터 |
| 남아메리카 | 12 | 남미 마스터 |

#### 4.3.3 대륙 정복자 (Continental Conqueror, v2 구현 범위)

권역(§4.3.2) "전부 정복" 조건은 평생 도전이라 v2에서는 보류하고, 대신 **각 대륙 안에서 일정 수 이상의 국가를 방문한 사용자**에게 "정복자" 칭호를 준다. 각 대륙별로 2단계(탐방가/정복자)를 둔다.

| 대륙 | 코드 | 탐방가 컷오프 | 정복자 컷오프 |
|---|---|---|---|
| 아시아 | AS | 5개국 | 12개국 |
| 유럽 | EU | 5개국 | 15개국 |
| 아프리카 | AF | 4개국 | 10개국 |
| 북아메리카 (중미·카리브 포함) | NA | 3개국 | 8개국 |
| 남아메리카 | SA | 3개국 | 7개국 |
| 오세아니아 | OC | 2개국 | 5개국 |
| 남극 | AN | — | 1개국(특수) |

명칭 예시:
- 탐방가: "아시아 탐방가 / Asia Wanderer", "유럽 탐방가 / Europe Wanderer", …
- 정복자: "아시아 정복자 / Asia Conqueror", "유럽 정복자 / Europe Conqueror", …
- 남극: "남극 탐험가 / Antarctica Pioneer" (방문 자체로 언락)

> 컷오프는 각 대륙의 국가 수(아시아 ~50, 유럽 ~50, 아프리카 ~54, 북미 ~23, 남미 ~12, 오세아니아 ~14)의 약 10%(탐방가) / 25~35%(정복자) 수준에서 라운드 넘버로 잡았다. 향후 4.3.2의 권역 분류가 정비되면 "전부 정복" 풀세트로 자연스럽게 확장된다.

#### 4.3.4 국가 단골 뱃지 (Per-Country Streak, v2 구현 범위)

특정 국가 한 곳을 오래/자주 다녀온 사람을 위한 뱃지. 국가별 누적 일수(`visit_days` 기준)로 3단계.

| 컷오프 | 명칭 (KR / EN) | 표시 예 |
|---|---|---|
| **30일** | (국가) 단골 / (Country) Regular | "일본 단골" |
| **100일** | (국가) 제2의 고향 / (Country) Second Home | "일본 제2의 고향" |
| **365일** | (국가) 토박이 / (Country) Local | "일본 토박이" |

- 뱃지 ID 형식: `country_<code>_d<threshold>` 예: `country_JP_d100`
- 본국이 자동 동기화 대상이 아니라는 점 때문에 보통 "국가 단골" 뱃지는 **해외 국가**에서 잠금 해제되며, 사용자가 본국 사진을 직접 추가했다면 본국에서도 잠금 해제될 수 있다.
- 국기 이모지로 시각화 (`🇯🇵 일본 토박이`).

#### 4.3.5 해외 사진왕 (Overseas Photographer, v2 구현 범위)

본국이 아닌 곳에서 찍은 사진의 절대 수량(`visit_photos` 중 `country_code != homeCode`)에 따른 4단계 뱃지. 국가 수와는 무관하게 "해외에서 얼마나 셔터를 눌렀나"를 측정.

| 컷오프 | 명칭 (KR / EN) | 의미 |
|---|---|---|
| **100장** | 해외 콜렉터 / **Overseas Collector** | 해외 사진 100장 |
| **500장** | 해외 마니아 / **Overseas Enthusiast** | 해외 사진 500장 |
| **1,000장** | 해외 마스터 / **Overseas Master** | 해외 사진 1,000장 |
| **3,000장** | 해외 전설 / **Overseas Legend** | 해외 사진 3,000장 |

- 본국 미설정 사용자에게는 모든 사진이 "해외"로 카운트되므로(보수적), 카운트는 본국 코드가 설정된 후 의미를 가진다.
- 일별 3장 제한(`addPhotos`)이 걸려 있어 한 번의 여행으로 무한 농사할 수 없다.

#### 4.3.6 사용자 활성 뱃지 (Active Title)

사용자는 잠금 해제한 모든 뱃지 중 **하나**를 "활성 뱃지"로 골라 홈 화면에 노출할 수 있다.

- 기본값: 현재 등급 뱃지(T1~UN_MASTER) — `getTierByCount(visited)`로 계산
- 사용자가 설정 화면에서 다른 뱃지를 고르면 그 뱃지의 명칭이 홈의 등급 라벨 자리를 차지
- 선택은 AsyncStorage에 영속 (앱 종료 후에도 유지)
- 새 뱃지가 잠금 해제될 때마다 토스트로 알림. 단, 첫 평가(=마이그레이션 직후)에는 묵음으로 누적

#### 4.3.7 특수 뱃지 (이후 확장 풀)

- **첫 발자국**: 처음 등록한 국가 (대개 본국)
- **첫 출국**: 본국이 아닌 첫 국가 — 별도 의미 부여
- **귀향**: 해외 여행 후 본국으로 돌아온 첫 사진
- **연속 방문**: 같은 국가 5회 이상
- **연도 정복**: 한 해 동안 N개국 신규 방문
- **계절 컬렉터**: 한 국가를 4계절 모두 방문
- **고도 수집가**: 5,000m 이상 고도에서 GPS 사진 1장
- **적도 횡단**: 적도 이남/이북 국가 모두 방문

## 5. 데이터 모델 설계 (개발 가이드)

### 5.1 등급 산정 로직

> 입력 `visitedCountries`는 **본국을 포함한 고유 국가 수**. 별도 제외 처리 없이 GPS 사진 기반 집합의 크기를 그대로 전달한다.

```
// 호출 예시:
//   const codes = new Set(Object.keys(activeCounts));
//   const tier = getTier(codes.size);

function getTier(visitedCountries: number): TierInfo {
  // 우선순위: 명예 등급 > 베이스 등급
  if (visitedCountries >= 193) return { tier: "UN_MASTER", prestige: 4 };
  if (visitedCountries >= 150) return { tier: "LEGEND", prestige: 3 };
  if (visitedCountries >= 100) return { tier: "GRANDMASTER", prestige: 2 };
  if (visitedCountries >= 75)  return { tier: "MASTER", prestige: 1 };
  if (visitedCountries >= 50)  return { tier: "T10" };
  if (visitedCountries >= 36)  return { tier: "T9" };
  if (visitedCountries >= 22)  return { tier: "T8" };
  if (visitedCountries >= 14)  return { tier: "T7" };
  if (visitedCountries >= 9)   return { tier: "T6" };
  if (visitedCountries >= 6)   return { tier: "T5" };
  if (visitedCountries >= 4)   return { tier: "T4" };
  if (visitedCountries >= 3)   return { tier: "T3" };
  if (visitedCountries >= 2)   return { tier: "T2" };
  if (visitedCountries >= 1)   return { tier: "T1" };
  return { tier: "T0" };  // 사진 0장 — 등급 없음
}
```

### 5.2 마일스톤(다음 등급) 표시 로직

홈 화면에서 "다음 마일스톤"은 현재 등급의 다음 컷오프 값을 사용.

```
const TIER_CUTOFFS = [1, 2, 3, 4, 6, 9, 14, 22, 36, 50, 75, 100, 150, 193];

function getNextMilestone(visited: number) {
  const next = TIER_CUTOFFS.find((c) => c > visited) ?? 193;
  return { next, remaining: next - visited };
}
```

> 기존 `App.tsx:43`의 `MILESTONES = [5, 10, 15, 25, 50, 100, 150, 195]`를 위 컷오프로 교체.
> `App.tsx:103-108`의 `totals.countries = Object.keys(activeCounts).length` 는 **그대로 유지** — 본국 포함 카운트가 그대로 등급 산정 입력값이 됨.

### 5.3 데이터 저장 구조 (제안)

type Achievement = {
  badgeId: string;          // "tier_t5", "region_middle_east", "days_100"

## 8. 결정 / 미해결 사항

### 결정됨

- **본국 카운트 정책**: 본국(homeCountry)도 등급 산정용 "방문 국가 수"에 **포함**한다. 본국 또한 "평생의 여행지"라는 철학에 따라, 그곳에서 찍은 GPS 사진이 1장 이상 있을 때 카운트된다. (§3.1, §3.2 참조)
- **컷오프 환산 규칙**: 외부 통계는 "해외 방문" 기준이므로 하위 등급(T1~T9)은 **+1** 환산하여 컷오프를 잡는다. 상위 절대 기준점(50/75/100/150/193)은 +1 대상 아님.

### 미해결 (Open Questions)

1. **백분위 표시 여부** — "상위 X%"를 노출하면 동기부여가 되지만, 정확한 인구 통계가 아닌 **추정치**임을 어떻게 알릴지 (각주? "추정")
2. **하향 등급 강등** — 사진 삭제로 국가 수가 줄어들면 등급도 내려갈지. **추천**: 등급 표시는 내려가되, 한번 잠금 해제된 뱃지(Achievement)는 영구 보존
3. **여행 일수 정의** — GPS 사진의 고유 일자 vs 첫 방문~마지막 방문 사이 모든 일자. 전자가 데이터 신뢰성 측면에서 우월
4. **여행 일수에 본국 포함 여부** — 국가 수와 달리 일수는 본국 거주일까지 포함하면 의미가 흐려짐. **추천**: 일수 뱃지는 **해외 체류일만** 집계 (국가 카운트와 정책 분리)
5. **명예 등급 컷오프 검증** — 75/100/150/193 외에도 TCC 회원 등급(Silver 150 / Gold 200 / Platinum 250)과 정렬할지. UN 가입국 193 vs ISO 195 vs TCC 리스트 330 — 어느 기준?
6. **본국 미설정 사용자 처리** — 본국이 미설정이어도 GPS 사진으로 카운트는 가능하나, "내 본국 발자취"라는 서사적 표현이 약해짐. 첫 사진의 GPS로 본국을 자동 제안할지 추후 결정.
