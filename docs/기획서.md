# VisitGrid 기획서

> 여행을 데이터처럼 쌓고, 시각적으로 수집하는 지도 앱

---

## 1. 개요

VisitGrid는 **도트(dot grid) 기반의 세계지도** 위에 여행 기록을 쌓아 올리는 앱입니다.
지도는 **각 나라를 대략적으로 식별할 수 있는 수준의 수백 개 도트**로 구성되며, 각 도트는 사용자의 방문 횟수에 따라 색의 농도가 짙어집니다.
기록은 반드시 **GPS 정보가 포함된 사진**으로만 등록할 수 있어, 데이터의 신뢰성을 확보합니다.

### 핵심 가치
- **수집의 즐거움**: 방문 도시·지역이 점점 진한 색으로 채워지는 시각적 만족
- **신뢰 가능한 기록**: GPS EXIF 검증으로 “직접 다녀온 곳”만 기록
- **데이터로서의 여행**: 여행을 점·선·면이 아닌, **격자 셀 단위 데이터**로 누적

### 한 줄 요약
> **여행을 “데이터처럼 쌓고 시각적으로 수집하는” 지도 앱**

## 2. 핵심 기능

### 2.1 도트 세계지도 (Dot World Map)
- 위경도 격자(grid)를 기반으로 한 도트 단위 세계지도
- **도트 수: 약 200~400개** (각 나라를 대략적으로 식별할 수 있는 수준)
  - 기본 격자 해상도: **8°** (육지 도트 약 ~300개)
  - 더 정밀하게 보고 싶을 때: 5° (≈ 1000개)
- 각 dot은 다음 정보를 포함:
  - `id`: 고유 식별자
  - `lat`, `lng`: 도트 중심 좌표
  - `countryCode`: 국가 코드 (ISO-3166-1 alpha-2)
  - `visitCount`: 방문 횟수 (Heatmap 입력값)

### 2.2 Heatmap 시각화
- `visitCount`에 따라 도트 색상이 단계적으로 진해짐
- 단계 예시:
  - 0회: 회색(미방문)
  - 1회: 옅은 색
  - 2~3회: 중간
  - 4회 이상: 진한 색
- 줌 레벨에 따라 dot 크기/밀도 조정 가능

### 2.3 여행 기록 (Travel Record)
- 사진 업로드 필수
- 사진의 EXIF에 **GPS 좌표가 반드시 포함**되어야 함
- 메모, 일자(촬영 일시 우선) 등 부가 메타데이터 저장 가능

### 2.4 위치 검증 (Verification)
- 사진의 EXIF GPS 좌표 → 속한 dot 계산
- 사용자가 선택한 dot과 일치할 때만 기록 저장
- 불일치 시 저장 차단 + 사용자에게 안내

## 4. 데이터 구조

### 4.1 DotCell
```ts
type DotCell = {
  id: string;          // "lat_lng" 형식 또는 인덱스
  lat: number;         // 도트 중심 위도
  lng: number;         // 도트 중심 경도
  countryCode?: string;
  isLand: boolean;     // 육지 여부 (지도 렌더링용)
  visitCount: number;  // 방문 누적 카운트
};
```

### 4.2 TravelRecord
```ts
type TravelRecord = {
  id: string;
  dotId: string;       // DotCell.id 외래키
  photoUri: string;    // 로컬 파일 경로
  lat: number;         // EXIF에서 추출한 좌표
  lng: number;
  takenAt: number;     // 촬영 시각 (epoch ms)
  createdAt: number;
  memo?: string;
};
```

### 4.3 SQLite 스키마(초안)
```sql
CREATE TABLE IF NOT EXISTS dot_cells (
  id TEXT PRIMARY KEY,
  lat REAL NOT NULL,
  lng REAL NOT NULL,
  country_code TEXT,
  is_land INTEGER NOT NULL,
  visit_count INTEGER NOT NULL DEFAULT 0
);

CREATE TABLE IF NOT EXISTS travel_records (
  id TEXT PRIMARY KEY,
  dot_id TEXT NOT NULL,
  photo_uri TEXT NOT NULL,
  lat REAL NOT NULL,
  lng REAL NOT NULL,
  taken_at INTEGER NOT NULL,
  created_at INTEGER NOT NULL,
  memo TEXT,
  FOREIGN KEY(dot_id) REFERENCES dot_cells(id)
);
```

## 5. 핵심 로직

### 5.1 도트 격자 생성
1. 위경도 그리드 해상도(예: 2°)를 정한다.
2. (-180~180, -90~90) 범위에서 격자 좌표를 생성한다.
3. 각 좌표가 **육지인지** 판정 → `isLand=true`인 도트만 저장한다.
   - GeoJSON(natural earth) + `@turf/turf`의 `booleanPointInPolygon`
4. 결과를 SQLite에 캐싱한다(최초 실행 시 1회).

### 5.2 사진 기록 플로우
1. 사용자가 dot 선택 → “기록 추가”
2. 사진 업로드 (`expo-image-picker`)
3. EXIF에서 GPS 추출 (없으면 거부)
4. 좌표 → dot 계산: `latLngToDotId(lat, lng, gridSize)`
5. 선택한 dot과 일치하면 저장 + `visitCount++`
6. 불일치하면 “선택한 위치와 사진의 좌표가 다릅니다.” 안내

### 5.3 좌표 → DotID 변환
```ts
function latLngToDotId(lat: number, lng: number, gridSize = 2): string {
  const dotLat = Math.floor(lat / gridSize) * gridSize + gridSize / 2;
  const dotLng = Math.floor(lng / gridSize) * gridSize + gridSize / 2;
  return `${dotLat.toFixed(2)}_${dotLng.toFixed(2)}`;
}
```

## 6. UX 흐름

### 6.1 메인 — 세계지도
- 전체 dot 분포 표시
- 줌/팬 제스처 (pinch/pan)
- Heatmap 색상으로 방문 분포를 한눈에

- dot 탭 → 모달/시트
- 해당 셀의 방문 기록 리스트(사진 썸네일 + 일시 + 메모)
- “기록 추가” 버튼

### 6.3 기록 추가
- 사진 선택 → EXIF 확인 → 검증 결과 표시

## 9. 미해결 / 결정 필요 항목

- 도트 해상도(gridSize): 기본 **8°** (육지 ~300개). 사용자가 줌인하면 5°/3°로 더 세밀하게 가변 처리 검토.
- 색상 팔레트: GitHub contribution 스타일 / OKLCH 그라디언트
- 지도 투영법: 단순 등각(equirectangular)부터 시작
- 향후 mercator / orthographic 지원 여부
